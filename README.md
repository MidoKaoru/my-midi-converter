# MIDI シャッフル解除ツール

シャッフルやハーフタイムシャッフルで打ち込まれたMIDIデータを、ストレート（イーブン）なリズムに変換するブラウザ完結型のWebアプリケーションです。

DAWで作成したシャッフル楽曲を、MuseScoreなどの楽譜作成ソフトへインポートし、読みやすい楽譜（ストレート記譜＋「Shuffle」指定）を作成する目的で使用します。

## 主な機能

* **リズムのイーブン化**: 揺れた発音タイミングを、最も近い正確なグリッド（8分音符または16分音符）にクオンタイズします。
* **デュレーションの補正**: 音符の長さ（ノートレングス）をグリッドの倍数に強制整形し、インポート時の細かい休符の発生を防ぎます。
* **連符の保護**: 3連符・5連符・7連符など、等間隔に並ぶフレーズを自動検出し、クオンタイズ処理をスキップして保護します。3連符は1拍内の音符間隔が PPQ/3 に近いかどうかで判定します。5音・7音以上の場合は音数で判定します。
* **2種類のリズム解釈**: 通常のシャッフル（8分音符ベース）と、ハーフタイムシャッフル（16分音符ベース）から選択可能です。小節単位で異なるモードを指定することもできます。
* **モード別デュレーション補正**: 8分音符ベースモードでは実質的な音符をすべて8分音符に補正します。16分音符ベースモードでは16分音符を基本単位として元の音価を最も近い倍数（16分・8分・付点8分など）に丸めます。
* **ブラウザ完結**: すべての処理はローカルのブラウザ内で完結し、外部サーバーへのデータ送信は行いません。

## 注意事項

* **装飾音符などの変形**: 装飾音符のような極端に短い音符は、処理によって強制的に指定のグリッド（8分音符または16分音符の長さ）に吸着・リサイズされるため、元の細かなニュアンスが失われたり変形したりします。これは楽譜上の複雑なゴミ休符やタイを防ぐための仕様です。装飾音符は、インポート後の楽譜ソフト上で手入力にて追加・修正することを前提としています。

* **移調楽器に関する注意**: サクソフォーンやトランペットなどの移調楽器を含むデータは、必ずお使いのソフト（DAW）で「実音（移調なし）」の状態にしてからMIDIエクスポートしてください。移調されたまま書き出すと、MuseScore等に読み込んだ際に音程や調号が狂う原因となります。

## 内部の処理フロー

本ツールでは、MIDIファイルの解析と変換において `@tonejs/midi` と `midi-file` の2つのライブラリを併用し、以下の手順で処理を実行しています。

これは、`@tonejs/midi` 単体で書き出しを行った際に発生する「調合（Key Signature）データの破損」および「マスタートラックと楽器トラックの意図しない分離」を回避するための実装です。

1. **キー情報の抽出 (`midi-file`)**
   元のMIDIファイルから `Key Signature`（調合）のイベントデータを抽出し、一時的に保持します。

2. **タイミングのクオンタイズ (`@tonejs/midi`)**
   `@tonejs/midi` を使用して、絶対時間（Ticks）ベースで音符のタイミングと長さのクオンタイズ処理を行います。処理の詳細は以下の通りです。
   * 曲中の拍子変更（Time Signature）を考慮し、小節ごとに正しい `ticksPerMeasure` を適用します。これにより、途中で拍子が変わる楽曲でも小節の区切りと連符検出が正確に機能します。
   * 各拍内の音符を分析し、3連符・5連符・7連符と判定されたフレーズはクオンタイズ対象から除外します。
   * 曲中のテンポチェンジ直上（±tolerance 以内）にある音符は発音タイミングをそのまま維持し、音価のみ補正します。ファイル冒頭のテンポ設定（tick=0）は保護対象から除外します。
   * 選択モードに応じてデュレーション補正を適用します（8分音符ベース：実質的な音符をすべて8分音符に統一 / 16分音符ベース：16分音符を基本単位として元の音価を最も近い倍数に丸め）。
   * この際、ライブラリ側の仕様によるメタデータの破損を防ぐため、元のキー情報等は一旦クリアした状態でバイナリデータとして出力します。

3. **データの修復と統合 (`midi-file`)**
   2で出力されたバイナリデータを `midi-file` で再度読み込み、以下のパッチ処理を行います。
   * 手順1で保持しておいた `Key Signature` 情報を Track 0 の先頭に書き戻します。
   * `@tonejs/midi` の仕様によって分離された Track 0（テンポマップ等）と Track 1（ノート情報）のイベントを、絶対時間に基づいてマージし、本来の一つのトラックに統合します。

4. **ファイル出力**
   修復と統合が完了したデータを、最終的なMIDIファイルとして出力・ダウンロードさせます。

## ディレクトリ構成

.
├── index.html       # UIとアプリケーションのエントリーポイント
├── css/

## 使用ライブラリ

* [@tonejs/midi](https://github.com/Tonejs/Midi) (via unpkg) - 時間軸のパースとクオンタイズ計算
* [midi-file](https://github.com/carter-thaxton/midi-file) (via esm.sh) - バイナリレベルでのメタデータ抽出・パッチ処理・トラック統合
